# Управление бэкапами PostgreSQL базы данных

Этот набор скриптов позволяет создавать и восстанавливать бэкапы PostgreSQL базы данных из Docker контейнера.

## 📁 Файлы скриптов

- `backup_db.sh` - Полнофункциональный скрипт для создания бэкапов
- `quick_backup.sh` - Быстрый бэкап без дополнительных проверок  
- `restore_db.sh` - Скрипт для восстановления базы данных из бэкапа
- `DATABASE_BACKUP_README.md` - Этот файл с инструкциями

## 🚀 Быстрый старт

### 1. Создание бэкапа (простой способ)
```bash
cd backend
./quick_backup.sh
```

### 2. Создание бэкапа (расширенный способ)
```bash
cd backend
./backup_db.sh
```

### 3. Восстановление из бэкапа
```bash
cd backend
./restore_db.sh ./backup_cocktails_db_2025-01-22_15-30-45.sql
```

## 📋 Подробное описание скриптов

### 🗃️ backup_db.sh - Полнофункциональный бэкап

**Особенности:**
- Создает директорию `./backups` если её нет
- Проверяет статус контейнеров перед бэкапом
- Создает символическую ссылку на последний бэкап
- Показывает размер созданного файла
- Выводит список последних бэкапов
- Цветной вывод для лучшей читаемости

**Использование:**
```bash
./backup_db.sh
```

**Создаваемые файлы:**
- `./backups/backup_{DB_NAME}_{TIMESTAMP}.sql` - файл бэкапа
- `./backups/latest_backup.sql` - ссылка на последний бэкап

### ⚡ quick_backup.sh - Быстрый бэкап

**Особенности:**
- Минимум проверок для максимальной скорости
- Создает бэкап в текущей директории
- Поддерживает передачу имени базы как аргумент
- Включает опции `--clean` и `--if-exists` для чистого восстановления

**Использование:**
```bash
./quick_backup.sh                    # Использует имя базы из .env
./quick_backup.sh my_database_name   # Указывает имя базы явно
```

**Создаваемые файлы:**
- `backup_{DB_NAME}_{TIMESTAMP}.sql` - файл бэкапа в текущей директории

### 🔄 restore_db.sh - Восстановление базы данных

**Особенности:**
- Запрашивает подтверждение перед восстановлением
- Показывает доступные бэкапы если файл не указан
- Проверяет статус контейнеров
- Выводит информацию о восстановленной базе

**Использование:**
```bash
./restore_db.sh ./backups/backup_cocktails_db_2025-01-22_15-30-45.sql
./restore_db.sh ./backup_cocktails_db_2025-01-22_15-30-45.sql
```

## ⚙️ Настройки

### Переменные окружения (.env файл)

Скрипты используют следующие переменные из `.env` файла:

```env
POSTGRES_DB=cocktails_db           # Имя базы данных
POSTGRES_USER=cocktails_user       # Пользователь PostgreSQL
POSTGRES_PASSWORD=your_password    # Пароль (не используется в скриптах)
POSTGRES_HOST=db                   # Хост (контейнер Docker)
POSTGRES_PORT=5432                # Порт PostgreSQL
```

### Значения по умолчанию

Если `.env` файл не найден, используются значения по умолчанию:
- **База данных:** `cocktails_db`
- **Пользователь:** `cocktails_user`
- **Docker Compose файл:** `docker-compose.local.yaml`
- **Сервис базы данных:** `db`

## 🐳 Требования

1. **Docker и Docker Compose** должны быть установлены
2. **Контейнеры должны быть запущены:**
   ```bash
   docker compose -f docker-compose.local.yaml up -d
   ```
3. **Файл docker-compose.local.yaml** должен существовать
4. **PostgreSQL контейнер** должен называться `db` в docker-compose файле

## 📊 Примеры использования

### Ежедневный бэкап
```bash
# Создать cron job для ежедневного бэкапа в 2:00
0 2 * * * /path/to/project/backend/backup_db.sh
```

### Бэкап перед обновлением
```bash
cd backend
./backup_db.sh
# Выполнить обновления...
```

### Восстановление после проблем
```bash
cd backend
# Посмотреть доступные бэкапы
ls -la ./backups/

# Восстановить последний бэкап
./restore_db.sh ./backups/latest_backup.sql
```

### Миграция данных
```bash
# На старом сервере
./backup_db.sh

# Скопировать файл на новый сервер
scp ./backups/backup_*.sql user@newserver:/path/to/project/backend/

# На новом сервере
./restore_db.sh ./backup_cocktails_db_2025-01-22_15-30-45.sql
```

## ⚠️ Важные замечания

1. **Восстановление полностью заменяет данные** в базе - будьте осторожны!
2. **Контейнеры должны быть запущены** перед выполнением скриптов
3. **Файлы бэкапов могут быть большими** - следите за местом на диске
4. **Пароль не требуется** благодаря настройке доверия в Docker контейнере
5. **Скрипты используют -T флаг** для неинтерактивного режима Docker

## 🆘 Устранение проблем

### Ошибка "контейнер не запущен"
```bash
docker compose -f docker-compose.local.yaml up -d
```

### Ошибка доступа к файлу
```bash
chmod +x backup_db.sh quick_backup.sh restore_db.sh
```

### Проверка статуса контейнеров
```bash
docker compose -f docker-compose.local.yaml ps
```

### Просмотр логов базы данных
```bash
docker compose -f docker-compose.local.yaml logs db
```

## 📈 Мониторинг размера бэкапов

```bash
# Просмотр размеров всех бэкапов
du -h ./backups/*.sql

# Удаление старых бэкапов (старше 30 дней)
find ./backups -name "backup_*.sql" -mtime +30 -delete
```
