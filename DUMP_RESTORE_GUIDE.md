# Руководство по восстановлению дампов базы данных

Этот документ содержит инструкции по использованию скриптов для восстановления дампов PostgreSQL в Docker контейнерах.

## Доступные скрипты

### 1. `restore_dump.sh` - Для локальной разработки

Скрипт для восстановления дампа в локальной среде разработки.

**Использование:**
```bash
./restore_dump.sh [путь_к_дампу] [docker-compose_файл]
```

**Параметры:**
- `путь_к_дампу` (по умолчанию: `archive/db_with_coctails.dump`)
- `docker-compose_файл` (по умолчанию: `docker-compose.local.yaml`)

**Примеры:**
```bash
# Использование с параметрами по умолчанию
./restore_dump.sh

# Указание конкретного дампа
./restore_dump.sh archive/my_dump.dump

# Указание конкретного docker-compose файла
./restore_dump.sh archive/db_with_coctails.dump docker-compose.dev.yaml
```

### 2. `restore_dump_prod.sh` - Для продакшн сервера

⚠️ **ОСТОРОЖНО!** Скрипт для продакшн среды с дополнительными мерами безопасности.

**Использование:**
```bash
./restore_dump_prod.sh [путь_к_дампу]
```

**Параметры:**
- `путь_к_дампу` (по умолчанию: `archive/db_with_coctails.dump`)

**Примеры:**
```bash
# Использование с параметром по умолчанию
./restore_dump_prod.sh

# Указание конкретного дампа
./restore_dump_prod.sh backup/production_dump.dump
```

## Что делают скрипты

### Общие шаги:
1. **Остановка сервисов** - останавливают web, celery-worker, celery-beat для освобождения подключений к БД
2. **Удаление старой БД** - полностью удаляют существующую базу данных
3. **Создание новой БД** - создают пустую базу данных
4. **Копирование дампа** - копируют файл дампа в контейнер PostgreSQL
5. **Восстановление дампа** - восстанавливают данные из дампа с флагами `--clean --if-exists`
6. **Очистка** - удаляют временные файлы
7. **Запуск сервисов** - перезапускают все остановленные сервисы
8. **Проверка данных** - выводят статистику восстановленных данных

### Дополнительные особенности продакшн скрипта:
- **Подтверждение пользователя** - требует явного подтверждения операции
- **Резервное копирование** - создает бэкап текущей БД перед восстановлением
- **Автоопределение контейнера** - автоматически находит контейнер базы данных
- **Восстановление при ошибке** - возможность восстановления из бэкапа при неудаче

## Устранение неполадок

### Ошибка: "Файл дампа не найден"
```bash
# Проверьте путь к файлу
ls -la archive/db_with_coctails.dump

# Или укажите полный путь
./restore_dump.sh /full/path/to/dump.dump
```

### Ошибка: "Контейнер базы данных недоступен"
```bash
# Проверьте статус контейнеров
docker compose -f docker-compose.local.yaml ps

# Запустите контейнер БД если он остановлен
docker compose -f docker-compose.local.yaml up -d db
```

### Ошибка: "Недостаточно прав"
```bash
# Сделайте скрипт исполняемым
chmod +x restore_dump.sh restore_dump_prod.sh
```

## Мониторинг процесса

Во время выполнения скрипты выводят подробную информацию:
- 🟢 **Зеленый** - успешные операции
- 🟡 **Желтый** - предупреждения
- 🔴 **Красный** - ошибки

## Логи и отладка

Для отладки можно использовать:
```bash
# Запуск с подробным выводом
bash -x ./restore_dump.sh

# Сохранение вывода в файл
./restore_dump.sh 2>&1 | tee restore_log_$(date +%Y%m%d_%H%M%S).txt
```
